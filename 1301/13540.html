On 5.1.2013 8:06, Leo Pilachowski wrote:
> I suspect one could write the code for a smart phone app to do the same
> thing. [the stroke watch]

I have actually done that for Palm handhelds. Currently the software
works as stopwatch, and with an extra button I can count the turns (the
number can be changed) and the software counts the RPM, plotting the
result on the screen (the left column for the time, the right for the
RPM); and the result can be stored in a database and retrieved from
there later. I have planned to do a PC link too, but actually have
thought that this application should double as the flight log, so I also
decided to add info columns on motor partials. Have not done that yet...

I think the application could (and should?) be transferred to Android.
Everything else is simple and straightforward, the only thing that
requires so,me tinkering is to find out the system calls to get the
timing data from the operation system. That probably is different for
Android than it is for Palm...
